# Практика по Python, 2026
Консольный индексатор файлов на Python с хранением данных в SQLite.

## Возможности

- Рекурсивное сканирование папок
- Фильтрация файлов по расширению, имени и размеру
- Поиск дубликатов по хэшу содержимого (SHA256)
- Сравнение папок с резервными копиями
- Хранение индекса и истории проверок в SQLite
- Кэширование хэшей для ускорения повторных запусков

## Требования

- Python 3.6 или выше
- Только стандартная библиотека (дополнительные пакеты не нужны)
- Windows: использовать `py` для запуска

## Запуск программы

Запуск происходит из корневой папки проекта:

```bash
cd light_python_practice_2026
py src/main.py <путь_к_папке> [команда] [опции]
```

## Команды

### Справка
```bash
py src/main.py --help
```
Показывает все доступные команды и опции.

### Сканирование папки
Сканирует папку и сохраняет метаданные файлов в базу данных.

```bash
# Все файлы без фильтров
py src/main.py <ваш путь> --scan --all

# Только Python файлы
py src/main.py <ваш путь> --scan --py-only

# Только текстовые файлы
py src/main.py <ваш путь> --scan --text-only

# Только медиа (изображения, аудио, видео)
py src/main.py <ваш путь> --scan --media-only

# Фильтр по расширениям
py src/main.py <ваш путь> --scan --ext .py .txt .md

# Фильтр по имени (содержит подстроку)
py src/main.py <ваш путь> --scan --name report

# Фильтр по размеру
py src/main.py <ваш путь> --scan --min-size 1024
py src/main.py <ваш путь> --scan --max-size 1048576

# Комбинация фильтров
py src/main.py <ваш путь> --scan --ext .py .txt --min-size 1000
```

### Поиск дубликатов
Находит одинаковые файлы по содержимому. Перед использованием нужно выполнить сканирование.

```bash
py src/main.py <ваш путь> --dups
```

### Сравнение с резервной копией
Сравнивает исходную папку с бэкапом, показывает изменённые, отсутствующие и лишние файлы.

```bash
py src/main.py <ваш путь> --backup <ваш путь>_backup
```

### История проверок
Показывает историю всех сравнений с бэкапами.

```bash
py src/main.py --history
```

## Что происходит при сканировании

1. Программа рекурсивно обходит все файлы и подпапки
2. Для каждого файла собирает: путь, имя, расширение, размер, дату изменения, тип
3. Пропускает скрытые файлы и папки (начинающиеся с точки)
4. Пропускает файлы без прав доступа
5. Сохраняет данные в таблицу `files`
6. При повторном сканировании: новые файлы добавляются, изменённые обновляются, удалённые помечаются как отсутствующие

## Что происходит при поиске дубликатов

1. Программа берёт список файлов из индекса
2. Делает быстрый анализ — находит файлы с одинаковым размером
3. Вычисляет SHA256 хэш содержимого (блоками по 64 КБ)
4. Если хэш уже считали раньше — берёт из кэша в БД
5. Группирует файлы с одинаковым хэшем
6. Показывает группы и сколько места можно освободить

## Что происходит при сравнении с бэкапом

1. Программа рекурсивно сканирует обе папки
2. Сравнивает файлы по относительным путям
3. Классифицирует различия:
   - `✓` Без изменений — файлы одинаковые
   - `~` Изменено — файл есть в обоих папках, но содержимое разное
   - `-` Отсутствует — файл есть в бэкапе, но нет в источнике
   - `+` Лишний — файл есть в источнике, но нет в бэкапе
4. Сохраняет результаты в таблицы `check_results` и `check_details`
5. Показывает подробный отчёт

## Типы файлов

Программа определяет тип файла по расширению:

| Тип | Расширения |
|-----|-----------|
| text | .txt .md .csv .log .py .js .html .css .json .xml .yaml .ini |
| image | .jpg .jpeg .png .gif .bmp .svg .ico |
| document | .pdf .doc .docx .xls .xlsx .ppt .pptx |
| archive | .zip .rar .7z .tar .gz |
| audio | .mp3 .wav .flac .aac |
| video | .mp4 .avi .mkv .mov |
| executable | .exe .dll .so .dylib |
| other | все остальные |

## Структура базы данных

База создаётся автоматически при первом запуске в папке `data/app.db`.

### Таблицы

| Таблица | Назначение |
|---------|-----------|
| `files` | Индекс файлов (путь, размер, дата, тип) |
| `file_hashes` | Хэши для поиска дубликатов (SHA256) |
| `check_results` | История проверок сравнения с бэкапами |
| `check_details` | Детали каждой проверки |


## Типичный сценарий использования

```bash
# 1. Первое сканирование папки
py src/main.py <Путь> --scan --all

# 2. Поиск дубликатов для экономии места
py src/main.py <Путь> --dups

# 3. Проверка синхронизации с бэкапом
py src/main.py <Путь> --backup <Путь бэкапа>

# 4. Просмотр истории проверок
py src/main.py --history
```
